Git & GitHub Tutorial

Como funciona e como utilizar?


O Git e o GitHub são pilares fundamentais que auxiliam as equipes de desenvolvimento a controlar o versionamento de código, rastrear mudanças, colaborar de forma eficiente e garantir que o trabalho em equipe flua sem problemas.


Git


O GIT é um sistema de controle de versão, também conhecido pela sigla VCS – version control system. Os VCS são uma ótima forma de otimizar o trabalho ao analisar as alterações feitas em um código de projeto compartilhado, além de ser um componente fundamental do sistema de gerenciamento de configuração de software que cuida das mudanças que precisam ser feitas em um projeto. As alterações/revisões/ atualizações feitas são identificáveis ​​através de códigos de letra ou números. Informações como data e a identidade do autor da alteração também são mantidas.


Git Qual a função? Como utilizar?
$ git init Inicializa um Repositório em um Diretório Existente. $ git init
$ git colne Utilizado para clonar um repositório. $ git clone <url do repositório>
$ git branch É uma bifurcação do estado do código que cria um novo caminho para a evolução do mesmo. Ou seja, gerencia branches (ramificações) no repositório $ git branch nova-branch
$ git checkout Muda entre branches ou restaura arquivos. $ git checkout nova-branch
$ git status Mostra o estado dos arquivos no repositório.
Exibe quais arquivos foram modificados, adicionados ou não monitorados.		 $ git status
$ git diff Mostra as diferenças entre versões de arquivos. Exibe as mudanças não comitadas. $ git diff
$ git add Adiciona o arquivo ao próximo commit. $ git add arquivo.txt
$ git commit Grava as mudanças no repositório. $ git commit -m "Mensagem do commit"
$ git push Envia commits para o repositório remoto. $ git push <repositório remoto> <branch>
$ git pull Puxa atualizações do repositório remoto e mescla. $ git pull <repositório remoto> <branch>
$ git revert Reverte um commit específico. $ git revert <hash_do_commit>
$ git merge Mescla branches diferentes. $ git merge nova-branch
$ git stash Armazena mudanças temporariamente. $ git stash
$ git rm Remove arquivos do repositório. $ git rm <arquivo>
$ config Configura opções do Git. $ git config --global user.name "Seu nome"
$ git reset Reseta o estado do repositório para um commit anterior. $ git reset --hard <hash_do_commit>
$ git remote Gerencia repositórios remotos. $ git remote add <repositório remoto> https://github.com/usuario/repo.git
$ git fetch Busca atualizações do repositório remoto. $ git fetch <repositório remoto>
$ git show Mostra informações sobre objetos do Git. $ git show <hash_do_commit>
$ git help Mostra ajuda sobre comandos do Git. $ git help <comando git>
$ git rebase Reaplica commits em cima de outro commit. $ git rebase <branch>
$ git blame Mostra quem modificou cada linha de um arquivo. $ git blame <arquivo>
$ git tag Marca pontos específicos na história do repositório. $ git tag <nome da tag>

GitHub


O GitHub é um serviço baseado em nuvem que um sistema de controle de versão (VCS) chamado Git. Ele permite que os desenvolvedores colaborem e façam mudanças em projetos compartilhados enquanto mantêm um registro detalhado do seu progresso


Aprendendo na Prática...


1° PASSO: Configurar o Git localmente


Se ainda não tem o Git instalado, baixe e intale a partir so site oficial do GitHub.


-> Configure o Git com duas informações de usuário:


$ git config --global user.name "Seu nome"
$ git config --global user.email "seuemail@exemplo.com"


2° PASSO: Criar um Repositório no GitHub


Crie uma conta no site oficial do GitHub.

No GitHub, clique no botão new para criar um repositório. Preencha o nome do repositório e uma descrição opcional. Escolha entre público ou privado. Clique no botão "Create Resopitory"


3° PASSO: Conectar o Repositório Local ao GitHub


Inicialize o repositório local (se ainda não feito); Pelo terminal navegue até o diretório do projeto e inicialize um novo repositório Git.


$ cd <caminho/do/seu/repositorio>
$ git init


Copie a URL do repositório recém-criado no GitHub (algo como https://github.com/usuario/repo.git). Adicione esta URL como um repositório remoto.


$ git remote add origin https://github.com/seu-usuario/seu-repositorio.git


4° PASSO: Fazer Commit e Push


Crie um arquivo README (opcional/e uma explicação do que é o projeto.)


$ echo "# Meu Projeto" >> README.md


Adicione os arquivos e faça um commit.


$ git add README.md
$ git commit -m "Adicionando arquivo README"


Defina o nome da Branch principal:


$ git branch -M main


Envie para o Repositório Remoto:


$ git push -u origin main


5° PASSO: Trabalhar com o Repositótio


Para obter as últimas mudanças do repositório remoto, use:


$ git pull origin main


Para enviar suas mudanças para o repositório remoto, use:


$ git push origin main


Tela do GitHub com os comandos



Seguindo estes passos, você configurará corretamente seu ambiente Git para trabalhar com repositórios no GitHub, podendo assim versionar e colaborar em projetos de forma eficiente.
